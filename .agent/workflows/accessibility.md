# 웹 접근성(Accessibility) 가이드라인

앞으로의 모든 디자인 및 코드 수정 작업 시 다음 규칙을 엄격히 준수합니다.

## 1. 색 대비 (Color Contrast) 규칙
- **일반 텍스트 (Normal Text)**: 배경색과의 명도 대비가 최소 **4.5:1** 이상이어야 합니다. (WCAG 2.1 AA 수준)
- **큰 텍스트 (Large Text)**: 대비 3:1 이상 (18pt 이상 또는 14pt 굵게 기준)
- **핵심 UI 요소 (UI Components)**: 단추, 아이콘 등 중요 인터랙션 요소는 배경과 3:1 이상의 대비를 유지해야 합니다.

## 2. 텍스트 가독성
- 흐린 회색(`#8E8E8E` 등)은 지양하고, `theme.colors.textLight`(`#6C6C6C`) 이상의 어두운 색을 사용합니다.
- 배경색이 파스텔 톤일 경우, 위에 올라가는 텍스트는 반드시 높은 대비를 가진 색상(예: `theme.colors.primary` 또는 `theme.colors.text`)을 사용합니다.

## 3. 상태 표시
- 색상만으로 정보를 전달하지 않습니다. (예: 오류 시 빨간색 테두리와 함께 텍스트 설명이나 아이콘을 병행)

## 4. 자동 검증 (향후)
- 새로운 컴포넌트 추가 시 명도 대비 체크를 수행합니다.
